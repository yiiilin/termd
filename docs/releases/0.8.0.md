termd 0.8.0

用户可见变化:
- Web 会话列表现在会显示 AI coding agent 的运行状态：运行中使用随主题适配的动态彩虹提示，需要操作、已完成和空闲状态使用清晰的静态标识。
- termd 在 daemon 侧从 PTY 输出和前台进程推导状态，支持 Codex、Claude Code、OpenCode 和 ZCode；客户端无需同时 attach 所有终端，也不会为活动状态新增高频 HTTP 轮询。
- AI 活动作为可选字段随现有 session metadata WebSocket 推送；旧客户端会忽略该字段，未识别的进程不会被猜测为 AI agent。
- 安装器回归测试的状态目录已强制隔离，并拒绝真实 `/var/lib/termd`、其子目录和符号链接等价路径，避免本机 QA 触碰运行中的 session 状态。

兼容性:
- supervisor IPC、双 WebSocket、JSON HTTP control、认证和 terminal frame 协议均未变化；`SUPERVISOR_VERSION` 仍为 `2026-07-12-dual-ws`。
- 从 0.7.2 升级到 0.8.0 不需要清理现有 session、终止 supervisor、重新配对设备或重新注册 daemon。AI 状态检测由新 daemon 消费现有 supervisor 输出完成。
- Web UI 嵌入在 `termd` 和 `termrelay` 二进制中；建议先升级公网 termrelay，再热更新本地 termd，并在升级前后核对 session ID、supervisor PID 和 socket 集合。
