termd 0.8.7

用户可见变化:
- Linux 预编译版的 `termd install`、`termrelay install` 和 `termctl install` 现在可以直接由下载后的单个二进制完成安装；安装器内置原先依赖外部工具的状态检查与配置处理，不再要求 Python、jq 或 sqlite3。
- README 安装步骤收敛为“下载二进制、执行 install”两步，同时提供简短的 curl 和 wget 命令；relay、daemon、termctl 的安装顺序、代理设置、升级和卸载入口集中在首页。
- 会话列表取消整卡跑马灯和独立拖动手柄；AI 状态改为卡片左侧的机器人工作、待处理或完成图标，直接拖动会话卡片即可排序，并用插入横线明确显示落点。普通点击仍会切换会话，键盘也可调整顺序。
- 会话切换时保留服务端已汇总的 AI activity，避免状态图标短暂闪烁或消失；折叠列表和展开列表使用同一份会话 activity。
- 没有 AI 状态的会话会根据 session ID 生成稳定的默认 identicon。图标库随 Web UI 本地打包，不使用 CDN，并为深浅主题提供统一的绿/青色系和浅灰边框。

兼容性:
- workspace、Cargo lockfile 和 Web 前端版本同步到 `0.8.7`；建议 daemon、Web UI、termctl 和 termrelay 同步升级。
- supervisor 兼容版本保持 `2026-07-12-dual-ws`；从 0.8.6 普通升级到 0.8.7 不需要替换 supervisor，也不会清空已有 session。
- 终端与 metadata WebSocket 协议未变化；已配对设备、relay 配置和现有 session 可继续使用。
- GitHub Release 仍只上传 Linux amd64/arm64 的 `termd`、`termctl`、`termrelay` 六个裸二进制。
