termd 0.8.5

用户可见变化:
- relay 安装完成后会创建或保留 setup token 与 daemon registry，并在受控终端中明确打印 setup token；安装 termd 时指定 `--relay` 会隐藏输入该 token，也可通过 `--relay-token` 或更安全的 `--relay-setup-token-file` 完成非交互安装。
- termd 安装器会把本机 daemon token 和 daemon public key 注册到 relay，relay 仅持久化 token hash，并按当前 `server_id` 验证 control 连接；成功后直接运行 `termd pair --qr` 输出二维码和配对邀请码，失败则明确返回非零状态及重试命令。
- relay 不再提供 open relay 兼容模式。termrelay 运行时必须配置非空 setup token 文件和 daemon registry，daemon connector 与浏览器请求都经过 trusted admission；relay 仍只负责 admission/routing，不持有 session 或 PTY 状态。
- Web 标题栏延迟改为在收到本次 metadata pong 后等待一秒再发起下一次检测；未响应的连接不会继续叠加 ping，显示值使用客户端时间戳计算真实往返延迟。
- 修复移动端软键盘打开时终端首尾行仍被强制居中而产生空白的问题：中间光标尽量居中，首行贴住会话标题区，末行贴住键盘上方，同时保持完整 PTY 网格、输入锚点和触摸滚动。

兼容性:
- open relay 配置不兼容 0.8.5；`--allow-open-relay` 已移除，直接运行 termrelay 时必须传入 `--setup-token-file` 与 `--daemon-registry`。使用 `termrelay install` 升级会创建或保留这两个 trusted 配置文件，不会继续暴露无鉴权入口。
- 为使用新的 trusted admission 和引导式注册，建议先升级 relay，再使用 relay setup token 升级或重新验证 termd。普通 termd 升级如果未显式修改 relay 地址，会保留已有 relay 配置且不强制重新输入 setup token。
- supervisor compatibility 仍为 `2026-07-12-dual-ws`；本版本不修改 supervisor IPC 或 session 持久化格式，正常升级不会清空现有 session、终止 supervisor、重新配对设备或更换 daemon identity。
