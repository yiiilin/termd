termd 0.8.3

用户可见变化:
- `termd`、`termrelay` 和 `termctl` 现在都自带 `install` / `uninstall` 子命令；安装前会分三步展示组件、配置和安全影响，支持无副作用的 `--dry-run`，默认交互确认。
- Linux amd64 release 新增稳定名称的原始二进制和统一 `checksums.txt`。用户可以校验下载文件后直接运行二进制完成安装，不再依赖先执行远程 `install-*.sh`；旧脚本继续作为 arm64 源码 fallback 和自动化兼容入口。
- 内嵌 installer 复用已有的 staging、回滚、systemd health 和 supervisor compatibility 检查。`--yes` 不会隐式授权清空 session；只有额外传入 `--allow-session-loss` 才允许不兼容 supervisor 升级移除会话。
- relay URL、代理 URL、token 文件和 TLS 私钥路径等敏感参数不会进入子进程 argv、Debug 输出或 dry-run 计划；installer 使用已打开的当前可执行文件完成自安装，避免安装期间路径被替换。
- README、安装指南和部署指南已改为逐步下载、checksum 校验、预览计划、安装和验证流程，并保留代理、arm64、升级与卸载说明。

兼容性:
- supervisor compatibility 仍为 `2026-07-12-dual-ws`；从 0.8.2 升级不会清空现有 session、终止 supervisor、重新配对设备或重新注册 daemon。
- terminal、metadata、JSON HTTP control、认证、relay routing 和 supervisor IPC 协议均未变化；0.8.3 可以与 0.8.2 的现有会话状态兼容。
- 建议 daemon、relay 和 CLI 同步升级，以便后续安装和升级统一使用内嵌 installer。Linux arm64 当前仍不提供预编译 binary，继续使用 release 脚本从源码构建。
