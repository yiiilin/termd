termd 0.8.10

用户可见变化:
- Web UI 新增标准 Web Push 后台通知；浏览器页面被冻结或关闭后，AI session 进入已完成、空闲或需要操作状态时仍可收到低频通知。
- 通知支持“仅需要操作”“全部状态”和关闭三种偏好；点击通知会恢复对应 daemon、打开 workspace 并 attach 到对应 session，页面可见时不会重复显示系统通知。
- 每个 daemon 使用独立的 Service Worker scope 和 VAPID identity；关闭通知或忘记 daemon 时会同时清理 daemon 订阅、浏览器 Push subscription 和对应 worker。
- 修复 terminal stream 因 supervisor heartbeat 超时或 slow-consumer 关闭后仍停留在假连接状态的问题；自动恢复和 `Refresh` 会丢弃失效 socket 并重新 attach，移动端与桌面端后台恢复都适用。
- Push 投递失败不会阻塞 daemon 启动、terminal、session 或 metadata 主链路；失效的 Push endpoint 会自动清理，通知内容不包含 PTY 输出、终端输入、access token 或私钥。

兼容性:
- workspace、Cargo lockfile 和 Web 前端版本同步到 `0.8.10`；使用 relay 时应同步升级 daemon、Web UI 和 termrelay，直连模式至少需要同步升级 daemon 与内嵌 Web UI。
- supervisor 兼容版本保持 `2026-07-12-dual-ws`；从 0.8.9 普通升级到 0.8.10 不需要替换 supervisor，也不会清空已有 session。
- Web Push 需要浏览器 secure context；`localhost` 和 `127.0.0.1` 可使用浏览器本地例外，普通 `http://192.168.x.x` 直连不支持。iOS/iPadOS 需要 16.4+、HTTPS，并将 Web 应用添加到主屏幕。
- 通知不依赖额外业务后端，但消息会经过浏览器厂商的 Push provider；daemon 仍持有订阅和 VAPID 私钥，relay 只做认证后的透明路由，不保存通知业务状态。
- SQLite 只新增 Web Push identity 和 subscription 表；旧版本会忽略这些表。GitHub Release 仍上传 Linux amd64/arm64 的 `termd`、`termctl`、`termrelay` 六个裸二进制。
