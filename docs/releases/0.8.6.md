termd 0.8.6

用户可见变化:
- `termd`、`termctl` 和 `termrelay` 新增 `upgrade` 子命令，可探测 GitHub 最新版本、交互确认并完成二进制替换与服务重启；下载过程会校验版本、架构、大小和 GitHub 提供的 SHA-256 digest，失败时保留或恢复原程序。
- `termd upgrade` 会单独处理 supervisor 兼容版本变化：普通确认不会授权丢失会话，只有用户明确允许 session loss 后才会执行不兼容的 supervisor 更新。
- GitHub Release 只上传 Linux amd64/arm64 的 `termd`、`termctl`、`termrelay` 六个裸二进制；不再上传项目生成的安装脚本、tar.gz 或 checksum 文件。
- 修复 metadata WebSocket 重连后延迟检测只发送少量 ping 就停止的问题；每次收到 pong 后等待一秒再发下一次测量，断线恢复后会继续运行。
- 客户端面板恢复可用，只显示当前在线客户端，并显示每个客户端正在查看的会话名称。
- AI 会话从运行中转为完成、待处理或空闲时可发送浏览器通知；会话列表中的 AI 活动边框加粗，更容易辨认当前状态。

兼容性:
- workspace、Cargo lockfile 和 Web 前端版本同步到 `0.8.6`；建议 daemon、Web UI、termctl 和 termrelay 同步升级。
- supervisor 兼容版本保持 `2026-07-12-dual-ws`；从 0.8.5 普通升级到 0.8.6 不需要替换 supervisor，也不会清空已有 session。
- `upgrade` 只接受 GitHub Release 中匹配当前 Linux 架构并带 SHA-256 digest 的正式二进制资产；不支持其他操作系统或未验证的下载地址。
- GitHub 自动生成的 Source code 归档仍会显示在 Release 页面，但不属于本项目上传的发布资产。
