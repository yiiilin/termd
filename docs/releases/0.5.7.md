# termd 0.5.7

## 用户可见变化

- 修复 Web 终端在直连或 relay 路径下偶发“闪一下像刷新、随后失去输入焦点”的问题。`outputResetVersion` 变化时，前端不再重建整棵 xterm DOM，而是原地 reset 当前终端实例，避免 helper textarea 被替换后浏览器直接丢焦点。
- full snapshot / resync 之后，新的 snapshot 和 live output 仍会按原有顺序继续消费；历史回放、scrollback reveal、移动端键盘态可视行缓存和首屏 reset 确认等待都补了回归覆盖，减少后续再出现“清屏修好了但输出链路坏了”的回归风险。
- 仓库默认忽略本地 `docs/superpowers/**` 协作草稿、`.codex-plan-*` 计划文件和 `not a tty` 噪音文件，减少本地调试/排查过程中把临时材料误带进正式提交的概率。

## 兼容性

- workspace / cargo / frontend 版本同步到 `0.5.7`。
- supervisor 兼容版本保持 `2026-06-10`；普通本地更新会保留已有 running session。
- pairing / invite 格式不变；已有配对设备和现有 session 不需要清空。
- relay 仍保持 dumb pipe，不解密、不解析业务明文。
