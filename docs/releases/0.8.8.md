termd 0.8.8

用户可见变化:
- Web 终端右上角的搜索入口替换为会话打开进度按钮。切换会话时会显示已完成步骤、当前步骤、下一步和各阶段耗时；打开完成后仍可点击按钮回看过程，失败时会保留失败阶段和重试提示。
- 进度按钮使用与状态匹配的图标：打开中显示旋转加载图标，完成后显示步骤清单，失败时显示错误图标。弹层会按终端区域自适应，在窄桌面、文件面板打开和移动端布局下不会被裁切或挡住终端操作。
- 会话列表默认头像改为随 Web UI 本地打包的 DiceBear `thumbs`，按 session ID 稳定生成，不请求外部头像服务。
- AI activity 改为头像右下角的状态角标，运行、待处理和完成状态不再替换会话头像；展开列表和折叠列表保持一致。
- Web 终端搜索入口及对应的前端搜索能力已移除。
- 修复重连旧式 supervisor 的会话在关闭响应随进程退出而丢失时误报 `Closing` 的问题；确认原 supervisor 进程已经退出后，关闭状态会正确收敛到 `Closed`。

兼容性:
- workspace、Cargo lockfile 和 Web 前端版本同步到 `0.8.8`；建议 daemon、Web UI、termctl 和 termrelay 同步升级。
- supervisor 兼容版本保持 `2026-07-12-dual-ws`；从 0.8.7 普通升级到 0.8.8 不需要替换 supervisor，也不会清空已有 session。
- 终端与 metadata WebSocket 协议未变化；已配对设备、relay 配置和现有 session 可继续使用。daemon 端既有搜索协议未删除，但 0.8.8 Web UI 不再提供搜索入口。
- GitHub Release 仍上传 Linux amd64/arm64 的 `termd`、`termctl`、`termrelay` 六个裸二进制。
