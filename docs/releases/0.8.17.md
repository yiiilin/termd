termd 0.8.17

用户可见变化:
- Web 工作台重新梳理视觉层级、深浅色语义和响应式布局：终端保持主工作面，窄桌面使用收起 rail，移动端使用覆盖面板，toolbar 在有限宽度下仍保留关键会话信息。
- 文件编辑器在关闭、按 Escape 或切换 session 时会保护未保存内容，并提供“保存 / 丢弃 / 留在此处”；关闭 session、删除文件、Git discard 和忘记 daemon 等不可恢复操作增加对象明确的确认。
- 移动快捷键设置改为草稿式“应用 / 取消”流程并提供逐行校验；弹窗、popover、触摸命中区、safe-area、页面缩放、减少动态效果和高对比偏好得到更完整支持。
- 文件与 Git 面板在局部请求失败时会保留已有内容，并提供重试或关闭错误提示，避免一次失败清空当前工作上下文。
- daemon 运行状态改为左对齐且按可用宽度分级展示；负载标签、CPU 数值不再被截断，宽屏 CPU 活动图增高增宽，窄屏优先保留终端空间。

兼容性:
- workspace、Cargo lockfile 和 Web 前端版本同步到 `0.8.17`。
- 本次变更仅涉及 Web UI、浏览器本地交互和相应测试，不改变 WebSocket 协议、pairing/auth、session 持久化、daemon/relay 路由或状态 schema；现有客户端与服务端接口保持兼容。
- supervisor 兼容版本保持 `2026-07-12-dual-ws`；从 0.8.16 普通升级无需替换 supervisor，也不会清空或终止已有 live session。
- GitHub Release 继续只发布 `termd`、`termctl`、`termrelay` 的 Linux amd64/arm64 六个裸二进制，不构建或推送容器镜像。
