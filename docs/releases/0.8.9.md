termd 0.8.9

用户可见变化:
- 修复移动端切到后台一段时间后返回时，终端已经断开却不再自动重连的问题；恢复到前台后会覆盖浏览器冻结且未执行的旧重连计时器，重新 attach 当前 session。
- 修复桌面端标签页或窗口长时间挂起后的同类问题；`visibilitychange`、`pageshow` 和窗口重新聚焦都会在连接确实失效时恢复终端，健康连接不会被重复重建。
- 连接错误提示中的 `Refresh` 现在也能清理卡住的旧重连状态并重新 attach；恢复期间输入会在连接建立后继续发送，不再停留在 `not attached` 状态。
- 恢复事件与已经开始的自动重连重叠时会复用当前 attach，不会把成功响应误判为过期连接，也不会重复 attach。

兼容性:
- workspace、Cargo lockfile 和 Web 前端版本同步到 `0.8.9`；建议 daemon、Web UI、termctl 和 termrelay 同步升级。
- supervisor 兼容版本保持 `2026-07-12-dual-ws`；从 0.8.8 普通升级到 0.8.9 不需要替换 supervisor，也不会清空已有 session。
- 终端、metadata WebSocket 和 relay 协议未变化；已配对设备、relay 配置和现有 session 可继续使用。
- GitHub Release 仍上传 Linux amd64/arm64 的 `termd`、`termctl`、`termrelay` 六个裸二进制。
