termd 0.8.14

用户可见变化:
- Git 工作区面板重新整理了仓库概览、文件变更和提交历史；变更数量、工作树状态和分支信息更容易扫读，Changes 与 Git graph 的分区高度也可以调整。
- Git 提交图加强了轨道、节点和当前 `HEAD` 的视觉层级，并且分别标记 `HEAD`、本地分支、远程分支和 tag，较长的提交信息仍会保持整齐。
- 修复 Changes 分区包含大量修改文件时无法纵向滚动、后续文件内容被容器裁切的问题；鼠标、触摸滚动和分区调整后都能继续查看完整列表。

兼容性:
- workspace、Cargo lockfile 和 Web 前端版本同步到 `0.8.14`；本次只调整内嵌 Web UI，不改变 daemon、relay 或 WebSocket 协议。
- supervisor 兼容版本保持 `2026-07-12-dual-ws`；从 0.8.13 普通升级不需要替换 supervisor，也不会清空已有 session。
- GitHub Release 继续只发布 `termd`、`termctl`、`termrelay` 的 Linux amd64/arm64 六个裸二进制，不构建或推送容器镜像。
