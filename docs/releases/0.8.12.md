termd 0.8.12

用户可见变化:
- 移动端终端新增贴合系统键盘的快捷键栏，默认提供 ESC、TAB、CTRL、ALT、SHIFT 和方向键；可展开 HOME/END、翻页、插入/删除、Ctrl、Fn 与符号面板。
- Ctrl、Alt、Shift 支持一次性粘滞组合，方向键支持长按重复；快捷键沿用 xterm.js 的输入链路，按当前 application cursor mode 为 Vim、tmux、less、htop 等 TUI 发送正确序列。
- iPhone Safari/PWA 和 Android Chrome/PWA 的横竖屏、键盘 viewport、safe-area、终端 fit 与焦点处理更稳定；系统键盘继续负责普通文字、中文输入法、语音和剪贴板。
- 兼容 iOS 中文输入法的 `,.?!` 轮换标点输入，并修复 iPhone 空格及大写字母因浏览器延迟事件造成的重复输入。
- AI 会话状态更醒目：运行中显示蓝色旋转边框和旋转图标，完成显示纯绿色常亮，等待操作显示黄色常亮，异常显示红色常亮；状态文字同步显示在会话行和折叠侧栏，用户启用减少动态效果时会停止旋转。
- 快捷键栏在 320px 窄屏下保持所有默认按键和文字完整可见，不遮挡终端最后一行。

兼容性:
- workspace、Cargo lockfile 和 Web 前端版本同步到 `0.8.12`；本次改动不改变 daemon、relay 或 WebSocket 协议。
- supervisor 兼容版本保持 `2026-07-12-dual-ws`；从 0.8.11 普通升级不需要替换 supervisor，也不会清空已有 session。
- GitHub Release 继续只发布 `termd`、`termctl`、`termrelay` 的 Linux amd64/arm64 六个裸二进制，不构建或推送容器镜像。
