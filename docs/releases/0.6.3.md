termd 0.6.3

用户可见变化:
- 修复 0.6.2 在本地 trusted devices 较多时，daemon control 重连会并发发起大量 relay device 注册请求，导致日志刷屏、请求超时甚至 `Too many open files` 的问题。
- relay 重启后的 device admission 恢复仍然保留，但 daemon 现在用单个后台任务顺序补注册本地 trusted devices，并汇总失败日志，避免注册风暴。

兼容性:
- 这是 0.6.2 的热修；不改变 pairing 流程、不改变 supervisor 兼容版本、不要求清空 session。
- 建议直接升级到 0.6.3，不要停留在 0.6.2。
