termd 0.6.2

用户可见变化:
- 修复 0.6.1 trusted relay 升级或重启后，已配对 Web 页面重连时报 `relay admission was rejected` 的问题。
- daemon control 重新连接 relay 后，会用本地持久化的 trusted devices 主动补注册 relay device admission；旧浏览器设备凭证不应因为 relay 重启而失效。
- relay 仍然不会放行未知设备：Web 设备重连时仍需要 device key 签名，daemon 只负责把自己已经信任的 device public key 重新声明给 relay。

兼容性:
- 这是 0.6.1 的 patch 修复；不改变 pairing 流程、不改变 supervisor 兼容版本、不要求清空 session。
- 建议把 daemon 升级到 0.6.2；relay 可以继续使用 0.6.1/0.6.2 的 trusted admission 接口。
