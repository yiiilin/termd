# termd 0.5.5

## 用户可见变化

- 修复 Web workspace 里终端 sidecar HTTP control 的瞬时失败升级成全局 `Connection error` 的问题。真实 relay 慢链路、快速切换持续输出 session，或浏览器短暂网络抖动时，`session.resize`、`session.cursor` 和 `session.list` 的旁路失败不再把当前 attach、metadata、files 和 status 一起拖死。
- 补回归测试覆盖 `session.resize` / `session.cursor` 的 HTTP control 瞬断，以及移动端空 workspace 手动刷新 `session.list` 的瞬时失败，确保这些错误只影响当次旁路请求，不会把页面打回 admin。
- 补强真实 relay 长串跑清理与噪声容忍：关闭已创建 session 时对页面已关闭/上下文已销毁场景做防护，并忽略预期的 `ERR_INCOMPLETE_CHUNKED_ENCODING`，降低长时间真实 relay 验证中的假红。
- 补强部分移动端和后台恢复 smoke 用例的超时预算，减少 release QA 下单 worker 串跑时的偶发时序误判。

## 兼容性

- workspace / cargo / frontend 版本同步到 `0.5.5`。
- supervisor 兼容版本保持 `2026-06-10`；普通本地更新会保留已有 running session。
- pairing/invite 格式不变；已有配对设备和现有 session 不需要清空。
- relay 仍保持 dumb pipe，不解密、不解析业务明文。
