# termd 0.5.1

## 用户可见变化

- 修复 Web 终端在新建 session、切换到其他 session 再切回后，首屏 prompt 或开头输出可能重复回显的问题。
- 修复 supervisor attach 同时携带 snapshot 和 tail frames 时，前端可能把 snapshot 已覆盖的 output 再写入一次的问题。
- 修复 tail-only attach 的序列游标播种：首条 tail frame 等于 `base_seq` 时不再误判为 gap 并触发不必要的 full snapshot。
- 修复文件面板关闭 Follow 后，迟到的静默 cwd 刷新或重新 attach 自动刷新可能把当前目录覆盖回 terminal cwd 的问题。
- 新建 session 的首屏输出现在会等 xterm reset 确认后再开始消费，减少切换和重连期间的旧实例写入竞态。

## 兼容性

- workspace / cargo / frontend 版本同步到 `0.5.1`。
- 该版本不改变 relay dumb pipe 模型、不改变 pairing/invite 格式，也不要求清空已有 supervisor session。
- 建议同时更新 `termd`、`termrelay` 和 Web 前端；混合版本仍走旧兼容路径，但 attach 首屏去重能力以新版 Web 为准。
