termd 0.8.2

用户可见变化:
- 修复移动端浏览器切到后台再返回后，终端可能停止输出、必须先输入内容才恢复的问题；页面恢复时会主动检查 terminal WebSocket，并在连接半开时自动重新 attach 当前 session。
- 延迟显示改为客户端发送毫秒时间戳、termd 原样回显后由客户端计算往返时间，不再显示固定或失真的 `1 ms`。
- Web UI 不再显示其他客户端的光标坐标和 focused/blurred 状态；operator 身份仍会正常显示，终端快照和 PTY 光标协议保持不变。
- 修复 release 安装脚本从管道执行时的路径解析和 systemd 环境问题，并补全 fresh install、升级、卸载与故障排查文档。
- `install-termd.sh`、`install-termrelay.sh` 和 `install-termctl.sh` 现在支持标准的 `http_proxy`、`https_proxy`、`all_proxy`、`no_proxy` 及对应大写环境变量，下载失败或 arm64 场景下的 Git、Cargo 和 npm 源码构建也会沿用代理。

兼容性:
- 双 WebSocket、JSON HTTP control、认证、terminal frame 和 supervisor IPC 保持兼容；`SUPERVISOR_VERSION` 仍为 `2026-07-12-dual-ws`。
- 从 0.8.1 升级不需要清理现有 session、终止 supervisor、重新配对设备或重新注册 daemon；建议 daemon、Web UI 和 relay 同步升级，以获得移动端恢复和准确延迟显示。
- terminal metadata ping 新增时间戳原样回显；未携带时间戳的旧客户端仍可继续使用原有 ping/pong 路径。
