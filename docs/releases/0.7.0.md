termd 0.7.0

用户可见变化:
- Web workspace 固定使用两条连接：metadata WebSocket 推送 session/client/status/RTT/CWD，terminal WebSocket 完成 create/attach、光标快照和持续 PTY 数据流；不再因创建、关闭或切换 session 触发 list/clients/cursor/resize HTTP 请求。
- 关闭当前 session 会立即从界面移除并关闭 terminal socket，只发送一次 JSON `session.close`；新建和 attach 从第一帧起即拥有终端连接，减少重复请求和偶发等待。
- 修复 relay 下新建 session 偶发等待约 4 秒的问题；Web 会复用已认证 workspace 连接，relay HTTP data 路由和 pairing deadline 也与双 WebSocket 协议保持一致。
- 修复快速切换 session 后终端可能不再输出的问题；旧 session 的 receive waiter 不会再取得并丢弃新 session 的首个 `attach_sync`。
- pairing 后持久保存 daemon 签名的 device certificate；设备私钥完成 challenge-response 后换取五分钟 access token，并提前 60 秒刷新。既有已配对设备通过受限 migration endpoint 换取 certificate，direct 和可信 relay 使用同一套认证链。
- application HTTP control 和错误响应统一为标准 JSON；文件上传 chunk 和下载字节流是仅有例外。旧 session token/scope、HTTP E2EE 和旧文件路径已移除。
- relay registry 只保存 daemon token hash 与 public key，用 public key 离线校验 daemon 签名的 credential；不注册或同步 pair ticket、device certificate、access token、session 或 PTY 状态。

兼容性:
- daemon、Web UI、termrelay 和 workspace package 版本同步到 `0.7.0`，不与 0.6.x 协议混跑。
- supervisor 兼容版本更新为 `2026-07-12-dual-ws`。升级顺序必须是 relay 先、termd 后；旧 relay registry 需要通过 root-only setup-token 文件重新注册 daemon public key。termd installer 会要求确认并清理旧 session supervisor，daemon identity、设备配对和普通配置会保留。
- relay 是可信 admission/routing 层，可见转发的应用明文；0.7 不存在运行时 E2EE 或 operator capability 层。
