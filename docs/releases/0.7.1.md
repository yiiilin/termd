termd 0.7.1

用户可见变化:
- 关闭当前 session 后，工作区会进入明确的无会话状态，不再自动选择或打开列表中的第一个 session；session 列表仍保持可用，用户可以自行选择后续要接入的终端。
- 关闭非当前 session 时不再关闭当前 terminal WebSocket，当前终端和操作者信息会继续保持，不会因为删除其他 session 而消失或中断输出。
- 修复快速切换 session 的合并窗口内关闭旧 session 时，新 session 的待执行 attach 被误取消的问题；切换目标仍会按计划建立 terminal WebSocket 并显示操作者。

兼容性:
- 0.7.x 双 WebSocket、JSON HTTP control 和认证协议保持不变；`SUPERVISOR_VERSION` 仍为 `2026-07-12-dual-ws`，从 0.7.0 升级无需清理现有 session、重新配对设备或重新注册 daemon。
- Web UI 嵌入在 `termd` 和 `termrelay` 二进制中；direct 与 relay 页面都需要使用 0.7.1 资源，建议先升级公网 termrelay，再升级本地 termd。
