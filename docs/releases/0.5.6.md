# termd 0.5.6

## 用户可见变化

- 为 `client -> relay -> termd -> supervisor` 链路补齐更完整的结构化诊断：前端会记录 direct/relay connect、`route_prelude`、handshake、HTTP request / first frame、queued payload 等 timeout；direct transport 还会记录 socket close/error 和 receive pump 结束摘要。配合 Web receive loop、relay route connect phase、daemon data pipe summary 与 supervisor request timeout 日志，可以更快判断“能输入但没有输出”是卡在 client、relay、termd 还是 supervisor。
- relay / daemon 的 timeout 与断流日志更细：route prelude、`route_ready`、control write、data pipe 首包有入站但始终没有任何输出、pending client 配对超时等场景都会留下明确 phase、timeout code、时长与关键标识，方便排查偶发 attach 超时或 relay 静默断流。
- 修复移动端 Web 终端输入恢复边界。触摸滚动终端时不再因为 `pointerdown` 提前弹出软键盘；点击终端正文时，即使 helper textarea 仍保留 DOM focus、但系统键盘已经收起，也会重新激活输入，恢复“点正文继续输入”的行为。
- 补回归测试覆盖 direct timeout diagnostics、direct handshake timeout、receive pump/socket close 摘要，以及移动端滚动不误弹键盘、点击正文重新激活输入等场景，降低后续回归风险。

## 兼容性

- workspace / cargo / frontend 版本同步到 `0.5.6`。
- supervisor 兼容版本保持 `2026-06-10`；普通本地更新会保留已有 running session。
- pairing / invite 格式不变；已有配对设备和现有 session 不需要清空。
- relay 仍保持 dumb pipe，不解密、不解析业务明文。
