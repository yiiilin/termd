termd 0.8.1

用户可见变化:
- 修复 `termctl 0.8.0` 连接 `wss://` daemon 或 relay 时，因 Rustls 同时编入多个 crypto provider 而在启动 TLS 前 panic 的问题。
- `termctl` 现在与 `termd`、`termrelay` 一样在进程入口显式选择 Rustls provider；真实公网 WSS 的 `list` 路径已使用 musl release 构建验证通过。
- 保留 0.8.0 的 AI coding agent 会话状态展示、安装器隔离和 Web UI 更新。

兼容性:
- 双 WebSocket、JSON HTTP control、认证、terminal frame 和 supervisor IPC 均未变化；`SUPERVISOR_VERSION` 仍为 `2026-07-12-dual-ws`。
- 0.8.0 的 `termd` 和 `termrelay` 与 0.8.1 客户端兼容；使用公网 `wss://` 的 `termctl 0.8.0` 应升级到 0.8.1。
- 从 0.8.0 升级服务端组件不需要清理 session、终止 supervisor、重新配对设备或重新注册 daemon。
