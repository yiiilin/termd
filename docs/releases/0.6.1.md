# termd 0.6.1

## 用户可见变化

- 修复移动端经 trusted relay 重新配对后，工作台会话列表可能显示为空且没有错误提示的问题。`session.list` 不再走 relay HTTP tunnel，而是复用已认证 WebSocket，避免首屏关键列表请求和 `daemon.status` 等旁路请求争抢 relay data pipe。
- 首次进入 workspace 的 `session.list` 失败不再被静默吞成假空态；后续已经稳定进入 workspace 的手动/后台刷新仍保持非致命，避免一次瞬时失败把用户打回管理页。
- 修复从 HTTPS relay Web 页面重新配对时，如果保存的旧 relay URL 与当前页面同 hostname 但端口或 path 已变化，Web 仍优先连接旧地址的问题。现在会优先当前页面派生的 `/ws` 地址，再回退旧保存地址。
- 移动端 Web UI 禁止页面级缩放，并把 workspace shell 固定到当前 `visualViewport` 的宽高和偏移；地址栏、软键盘或浏览器 viewport 变化后，终端窗口更稳定地铺满当前可视区域，减少底部或侧边空白。
- 移动端状态栏、快捷输入栏和 sessions/files 面板的宽度回归覆盖已同步到新的 visual viewport 布局。

## 兼容性

- workspace / cargo / frontend 版本同步到 `0.6.1`。
- 0.6.1 是 0.6.x trusted relay 模型的补丁版本；仍要求 daemon、relay、Web UI 和 `termctl` 同步使用 0.6.x，不支持和 0.5.x 不可信 relay/E2EE 运行时模型混用。
- 本次升级不改变 daemon registry、daemon token、device pairing 状态和 supervisor 兼容版本；已有 running session supervisor 不需要清空。
