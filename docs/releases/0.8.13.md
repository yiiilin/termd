termd 0.8.13

用户可见变化:
- 通过代理、SSH 转发或公网链路接入会话时，如果 daemon data pipe 的 WebSocket 建连偶发卡住，termd 会在 relay 的配对窗口内自动重试，减少浏览器长时间停在“正在接入会话”的情况。
- data pipe 单次连接尝试最长 4 秒，并按 500ms、1s、3s、10s 退避；完整 WebSocket、route hello 和 route ready 握手共享 20 秒总截止时间，不会在 relay 已结束等待后继续留下过期连接任务。
- 仅在发送 route hello 前的连接超时时重试；认证拒绝、协议错误、发送失败和 route ready 超时仍立即返回，避免掩盖不可恢复的问题。

兼容性:
- workspace、Cargo lockfile 和 Web 前端版本同步到 `0.8.13`；本次修复不改变 daemon、relay 或 WebSocket 协议。
- supervisor 兼容版本保持 `2026-07-12-dual-ws`；从 0.8.12 普通升级不需要替换 supervisor，也不会清空已有 session。
- GitHub Release 继续只发布 `termd`、`termctl`、`termrelay` 的 Linux amd64/arm64 六个裸二进制，不构建或推送容器镜像。
