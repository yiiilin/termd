termd 0.8.16

用户可见变化:
- relay 接入会话时，daemon data pipe 的单次建连超时依次使用 `500ms`、`1s`、`3s`、`10s`；前几次可以更快放弃已卡住的连接，后续尝试仍为弱网保留更长时间。
- 单次建连超时后会立即进入下一档重试，不再另外等待固定退避间隔，减少 relay 丢包或短暂抖动时接入会话长时间停顿。

兼容性:
- workspace、Cargo lockfile 和 Web 前端版本同步到 `0.8.16`；本次只调整 daemon 到 relay 的 data pipe 建连超时与重试策略，不改变 WebSocket 协议或路由格式。
- supervisor 兼容版本保持 `2026-07-12-dual-ws`；从 0.8.15 普通升级不需要替换 supervisor，也不会清空已有 session。
- GitHub Release 继续只发布 `termd`、`termctl`、`termrelay` 的 Linux amd64/arm64 六个裸二进制，不构建或推送容器镜像。
