# termd 0.2.0 Protocol Implementation Plan

> 历史状态提示：本文记录当时的计划/实现状态，不代表当前 0.6 协议契约；现行边界以 `TECH.md` 和 `docs/deployment.md` 为准。

> **For agentic workers:** REQUIRED: use subagent-driven development where work can be safely split. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the ad-hoc inner message protocol with a complete 0.2.0 packet layer that supports unary RPC, streams, cancellation, errors, flow control, resumable stream windows, and direct/relay consistency.

**Architecture:** Keep WebSocket and relay as transport only. Keep E2EE as the privacy boundary, but put a new `ProtocolPacket` envelope inside encrypted frames and route all user-visible operations through method names, request ids, stream ids, sequence numbers, and credit. The daemon uses existing business handlers as the first migration adapter, while Web/CLI move to packet APIs.

**Tech Stack:** Rust 2024, serde JSON wire types, Axum WebSocket transport, TypeScript/Vite frontend protocol client, existing X25519/HKDF/ChaCha20-Poly1305 E2EE.

## Execution Rule

Progress must be tracked in this file only:

- Start every pending task as unchecked
- Mark each task with `[x]` immediately after implementation and verification complete
- If a task reveals a prerequisite gap, add a new unchecked task directly below it before continuing
- If any task remains unchecked, the project is not complete

## Scope

- Packet version is `3`; product/crate release target is `0.2.0`.
- Backward compatibility is not a product requirement, but old daemon handlers may remain as internal migration adapters until all clients use packet APIs.
- Relay remains a dumb pipe and must not parse decrypted business packets.
- The implementation must be covered by Rust unit/e2e tests and frontend protocol tests before completion is claimed.

## Tasks

- [x] **Task 1: Define protocol packet wire types**
  - Files: `proto/src/lib.rs`, `termui/frontend/src/protocol/types.ts`
  - Add request id, stream id, packet kind, packet envelope, packet error, stream credit/window fields, and method constants where useful.
  - Add serialization tests for request, response, stream chunk, stream end, cancel, flow, event, and error packets.

- [x] **Task 2: Bind E2EE handshake to daemon identity**
  - Files: `proto/src/lib.rs`, `termd/src/auth/mod.rs`, `termd/src/net/protocol.rs`, `termui/frontend/src/protocol/e2ee.ts`, `termctl/src/client.rs`
  - Add signed daemon E2EE key exchange material and client verification before any pair/auth/session packet is sent.
  - Ensure auth signatures bind the current E2EE transcript enough to prevent relay challenge forwarding.

- [x] **Task 3: Implement daemon packet dispatcher**
  - Files: `termd/src/net/protocol.rs`, `termd/src/net/server.rs`, `termd/src/net/relay.rs`
  - Add packet request/response dispatch, packet error mapping, stream registry, stream sequence counters, cancel handling, and flow-credit accounting.
  - Map existing business handlers to method names without putting business logic in relay.

- [x] **Task 4: Migrate Web protocol client to packet dispatcher**
  - Files: `termui/frontend/src/protocol/direct-client.ts`, `termui/frontend/src/test/mock-daemon.ts`, frontend protocol tests
  - Add a receive pump that resolves responses by request id and queues stream/event packets separately.
  - Convert unary methods to `request(method, payload)`.
  - Convert terminal attach/output/input to a stream with stream id, chunk sequence, cancellation, and credit replenishment.

- [x] **Task 5: Migrate termctl to packet APIs**
  - Files: `termctl/src/client.rs`, `termctl/src/cli.rs`, `termctl/tests/direct_daemon_e2e.rs`
  - Convert pair/auth/session/list/control/resize/ping to packet methods.
  - Convert interactive terminal attach to a terminal stream.

- [x] **Task 6: Harden transport boundaries**
  - Files: `termd/src/net/server.rs`, `termd/src/net/relay.rs`, `termrelay/src/ws.rs`
  - Add route prelude timeout, connect/route/send/pong deadlines, idle timeout, and maximum frame size for direct and relay paths.
  - Keep relay behavior opaque to business packet contents.

- [x] **Task 7: Verify and version 0.2.0**
  - Files: crate manifests, `package.json`, docs/release notes as needed
  - Run `cargo test --workspace --locked`.
  - Run frontend typecheck, unit tests, and production build.
  - Update versions to `0.2.0` only after the protocol tests pass.
